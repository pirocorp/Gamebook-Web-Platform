# ADR-0011: Support media assets from the start

## Status

Accepted

## Context

The platform should support images, audio, and maps from the beginning.

## Decision

Add media asset support in the first version.

Supported initial media categories:

- Images.
- Audio.
- Maps.

## Reasons

- Many gamebooks benefit from maps and illustrations.
- Designing media later would require changes to episodes and editor workflows.
- Media support improves presentation and future book variety.

## Consequences

Positive:

- Richer reading experience.
- Future-friendly content model.

Negative:

- Requires upload handling.
- Requires storage configuration.
- Requires media validation and metadata.
